EV Learning Kit
Python RAG FastAPI Next.js PostgreSQL AWS Claude API OpenAI Embeddings
View on GitHub →
Goal

Build a RAG-based application that answers questions about electric vehicles and generates realistic DC charge curves for any vehicle class, combining a curated knowledge base with an LLM-driven curve engine backed by real reference profiles.

Problem

When I first joined the EV industry, I struggled to build a solid foundation of knowledge. Technical concepts around battery chemistry, charging behaviour, and powertrain systems were scattered across dozens of sources with inconsistent quality, and there was no easy way to see how charge curves actually behave across different vehicle classes and conditions. I wanted to create something I wish I had when I started: a single platform that pools trusted sources and lets users interact with the concepts directly rather than just reading about them.

Key Work
  • Built an EV Q&A system that retrieves relevant knowledge chunks from a curated knowledge base and generates grounded answers via Claude, reducing hallucination risk through retrieval augmentation.
  • Built a Charge Curve Generator that takes vehicle class, battery size, and max DC power as inputs and produces a native charge curve alongside a delivered curve clipped to the site charger capacity, with supporting metrics and reasoning.
  • Built an incremental RAG ingestion pipeline using OpenAI embeddings stored in PostgreSQL via pgvector, sourcing from curated EV articles and technical PDFs, with only new content embedded on each run.
  • Deployed the knowledge base on AWS RDS with credentials managed via Secrets Manager and exposed both capabilities via a FastAPI backend with a Next.js frontend for interactive use.
Impact
  • Produces grounded, cited answers to EV questions, reducing hallucination risk through retrieval augmentation.
  • Enables users to simulate realistic charging behaviour for different vehicle classes.
Chargepoint Load Analysis
Python Machine Learning XGBoost Streamlit ETL
View on GitHub →
Goal

Use real-world EV charging session data from Scotland's Chargepoint network to identify where additional charging infrastructure is needed most, supporting data-driven decisions for EV infrastructure planning.

Problem

As EV adoption grows, understanding where charging demand exceeds capacity is critical for infrastructure planning. Identifying high-pressure regions from session-level data requires feature engineering, temporal analysis, and a classification model that can flag congestion-risk areas before they become bottlenecks.

Key Work
  • Built an ETL pipeline to harvest session data from the Smart Energy Data Service API, clean raw records, and extract time features (hour, day, month).
  • Defined a binary classification target — needs more chargers (1) vs sufficient capacity (0) — based on utilisation rate thresholds and peak demand analysis.
  • Trained and evaluated Logistic Regression, Random Forest, and XGBoost models using accuracy, precision, recall, F1, and ROC-AUC.
  • Produced a heatmap of infrastructure pressure across Scotland using GeoPandas and Folium, and built a Streamlit dashboard for interactive exploration.
Impact
  • Produced a ranked list of high-priority regions for new charging infrastructure, directly actionable for infrastructure planners.
  • Surfaced temporal demand patterns (peak hours, weekday vs weekend usage) to inform charger deployment timing.
